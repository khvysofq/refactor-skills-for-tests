> **前置条件**：无（或从 [Workflow.md](../Workflow.md) 决策树 Q1 为「否」进入）  
> **目标**：形成工程目标、输入/输出、主流程的概览，并写入总体报告

- 或需补齐总体报告中的「工程概览」部分

## 执行指令

**加载 Skill**：阅读并执行 → [Skill 01: 工程概览与主流程](../skills/skill-01-overview.md)

## 阶段验收标准

在 Skill 01 完成后，验证以下条件全部满足：

### 必须满足

- [ ] **总体报告存在**
  ```bash
  [ -f docs/codearch/overall_report.md ] && echo "PASS" || echo "FAIL"
  ```

- [ ] **工程概览完整**：文档包含以下四块且内容非空
  - [ ] 工程目标（或等价标题）
  - [ ] 输入
  - [ ] 输出
  - [ ] 主流程

- [ ] **技术特征概览完整**：文档包含以下内容
  - [ ] 语言与标准（主要语言、语言标准）
  - [ ] 入口点索引（至少 main() 位置）
  ```bash
  grep -q "技术特征概览\|语言与标准" docs/codearch/overall_report.md && echo "PASS" || echo "FAIL"
  grep -q "入口点索引\|main()" docs/codearch/overall_report.md && echo "PASS" || echo "FAIL"
  ```

- [ ] **验收检查通过**（见 Skill 01 中的验收检查命令）

### 可选

- [ ] 主流程以步骤列表或 Mermaid 图形式呈现，便于后续引用
- [ ] 技术特征统计已有初步估计（待 Phase 02 完成后更新）
