# Skill 06: 报告汇总与验收

> **触发**: Phase 06 指示  
> **目标**: 汇总总体报告、确认所有报告引用正确  
> **预计耗时**: 15–60 分钟

---

## 输入

- P0–P5 各阶段的产出
- [产出路径与报告结构约定](../definitions/output_structure.md)

## 输出

| 产出物 | 路径 | 必须 |
|---|---|---|
| 总体报告（最终版） | `docs/codearch/overall_report.md` | ✅ |

---

## 核心任务

### 任务 1: 汇总总体报告

确保 `docs/codearch/overall_report.md` 包含全部必填章节，参照 [总体报告模板](../templates/overall-report.md)：

1. **工程概览**：工程目标、输入、输出、主流程
2. **技术特征概览**：语言与标准、技术特征统计、入口点索引
3. **模块索引**：模块列表（含增强结构）、模块依赖图（Mermaid）、跨模块关系摘要
4. **构建与测试摘要**：简要说明或链接到 `build_and_tests.md`
5. **信息来源汇总**：分析过程中发现并利用的文档与信息源

若已有部分内容（来自前序阶段），保留并补全；若缺失，根据已有产出补写。

### 任务 2: 模块索引完整性

验证模块索引的结构完整性：

- 模块列表表格包含状态列（叶子 / 已拆分）
- 已拆分模块的子模块以 `↳` 缩进展示
- 模块依赖图为 Mermaid 格式，含依赖类型标注
- 枢纽模块（≥3 个依赖方）标注 `[HUB]`
- 跨模块关系摘要为 2–3 段描述性文字

### 任务 3: 引用校验

确保总体报告中的所有模块链接指向存在的 L2 报告：

```bash
grep -oE 'modules/[^)]+\.md' docs/codearch/overall_report.md | sort -u | while read p; do
  [ -f "docs/codearch/$p" ] && echo "OK $p" || echo "BROKEN $p"
done
```

若发现断裂链接，修正链接或标记缺失报告。

### 任务 4: L2 报告完整性抽查

随机或按优先级抽查 2–5 份模块报告，确认包含 L2 必填章节：

- 职责描述
- 边界（输入/输出）
- 依赖
- 复杂度评级
- 关键设计要点
- 代码特征
- 关键代码位置索引
- 关键数据流路径
- 与其它模块的关系

缺项则退回 P4 补充。

### 任务 5: 技术特征统计

验证总体报告的「技术特征统计」表格与各模块报告的代码特征一致且为最新状态：

- 逐项核对：手动内存管理、多线程/并发、文件 I/O、网络 I/O、外部数据处理
- 确认涉及模块数与主要模块列表准确

---

## 验收标准

- [ ] `docs/codearch/overall_report.md` 存在且包含全部必填章节
- [ ] 总体报告中所有模块链接指向存在的文件（无断裂链接）
- [ ] 模块索引含状态列与层次缩进
- [ ] 模块依赖图（Mermaid）存在且含依赖类型标注
- [ ] 抽查的 L2 报告均包含必填章节
- [ ] 技术特征统计为最新状态

### 验收检查

```bash
[ -f docs/codearch/overall_report.md ] && echo "PASS" || echo "FAIL"
grep -q "modules/.*\.md" docs/codearch/overall_report.md && echo "PASS" || echo "FAIL"
grep -q "mermaid" docs/codearch/overall_report.md && echo "PASS" || echo "FAIL"
```

---

## 注意事项

| 类别 | 说明 |
|---|---|
| **DO** | 以 [output_structure](../definitions/output_structure.md) 为参考标准 |
| **DO** | 链接使用相对路径，便于不同环境打开 |
| **DON'T** | 不在总体报告中内嵌模块报告全文（仅保留链接以支持按需加载） |
| **DON'T** | 不删除已有的模块报告文件 |

## 跳转

| 结果 | 下一步 |
|---|---|
| 验收通过 | 代码认知阶段完成，可进入阶段二（风险评估） |
| 未通过 — 链接断裂 | 修正链接或补建缺失报告后重新验收 |
| 未通过 — 章节缺失 | 补写总体报告章节后重新验收 |
| 未通过 — L2 报告不完整 | 退回 P4 补充后重新验收 |
