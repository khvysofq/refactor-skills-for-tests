# 收敛条件与迭代控制

> **阅读时机**：P3 收敛评估时阅读。

---

## 目的

每完成一轮 P1→P2 迭代后，评估当前模块分解是否已收敛。若已收敛，进入 P4 深入分析；若未收敛，产出**再入列表**——列出需要重新进入 P1 进行进一步分解的模块。

---

## 收敛条件

一个**叶子模块**满足以下任一条件即视为已收敛：

| 条件 | 说明 |
| ---- | ---- |
| **(a) 复杂度 ≤ 中** | 无需进一步拆分 |
| **(b) 复杂度为高/极高，但持有合格的「不拆」理由** | 理由必须包含具体技术论证（跨模块依赖、双向耦合、共享关键状态），详见下方[「不拆」理由格式要求](#不拆理由格式要求) |
| **(c) 递归深度已达上限** | 默认上限 3 层：顶层=1，首次拆分=2，再次拆分=3 |

**整体收敛判定**：当所有叶子模块均满足上述条件之一时，本轮迭代收敛。

---

## 迭代控制规则

| 规则 | 说明 |
| ---- | ---- |
| **最大迭代轮数** | 3 轮（从首次执行 P1 开始计数） |
| **达到上限的处理** | 将剩余高复杂度模块标注为「已达迭代上限」，直接进入 P4 |
| **进度要求** | 每轮迭代必须产生进展——至少一个模块完成拆分或获得合格的「不拆」理由 |
| **无进展强制收敛** | 若某轮迭代未产生任何进展，强制判定收敛，进入 P4 |

---

## P3 评估维度

对当前模块划分逐项检查：

| 维度 | 检查要点 |
| ---- | -------- |
| **叶子模块收敛** | 所有叶子模块均满足收敛条件 (a)/(b)/(c) |
| **拆分说明完整** | 所有触发拆分评估的高/极高模块均已标注终态拆分状态（「已拆」或「不拆」），不允许「待拆」或含糊表述残留 |
| **粒度一致** | 同层级模块之间不存在体量悬殊的情况 |
| **评级交叉验证** | 复核 P2 评级——是否存在客观指标应导致更高等级但实际评级偏低的模块（反偷懒检查） |
| **索引一致** | 总报告中的模块索引与磁盘上实际存在的模块报告一一对应 |

**结论**：全部通过 → **已收敛**；任一项不通过 → **未收敛**，须产出再入列表。

---

## 未收敛时的处理

产出**再入列表**，明确每个未收敛模块的处置方式：

| 情况 | 处置 |
| ---- | ---- |
| 拆分说明为「待拆」 | 进入 P1，识别子模块 |
| 复杂度评级疑似偏低 | 重新进入 P2，按客观指标重新评级 |
| 模块报告缺失 | 进入 P1，生成对应模块报告 |

多个模块可在 P1 中并行处理。

## 不拆理由格式要求

当高/极高复杂度模块决定不拆分时，「不拆」理由**必须**包含以下至少一项具体技术论证：

1. **拆分将引入新的跨模块依赖**——说明拆分后哪些内部调用会变为跨模块调用，以及由此引入的依赖路径。
2. **内部存在双向耦合，无法划出干净边界**——指出具体的双向调用链或相互引用关系。
3. **共享关键内部状态，拆分后需暴露该状态**——说明哪些内部状态会被迫成为公开接口，以及带来的风险。

**以下理由不合格，不能作为「不拆」的充分依据**：

- ❌「各子模块职责清晰」
- ❌「保持完整性」
- ❌「通过接口解耦」
- ❌「已有子目录划分」
