# Phase 03: 收敛评估

> **阶段代号**: P3  
> **入口条件**: P2 已完成  
> **预计耗时**: 每轮评估 15–30 分钟

---

## 目标

评估所有模块是否已收敛——即所有叶子模块的复杂度 ≤ 中（且符合地板规则），或已给出合理的不拆分理由，且所有「已拆」声明均有实际子模块报告文件支撑。

---

## 执行

加载并执行 [Skill 03 — 收敛评估](../skills/skill-03-converge.md)。

参考定义：
- [convergence_criteria — 收敛判定标准](../definitions/convergence_criteria.md)

---

## 核心决策逻辑

```
STEP 1: 执行自动化验证门禁（Skill 03 Task 0）
  - Check A: 地板规则合规性（脚本检查）
  - Check B: 已拆报告存在性（脚本检查）

IF Task 0 检测到 VIOLATION 或 MISSING:
  → 未收敛（自动化门禁未通过），产出再入列表
  → 地板规则违规 → 重入 P2
  → 子模块报告缺失 → 重入 P1
  （不执行后续检查）

STEP 2: 执行主观评估（Skill 03 Task 1–4）

IF 所有叶子模块已收敛:
  → 进入 P4（深度分析）
ELSE:
  → 生成再入列表
  → 对需要进一步分解的模块重新进入 P1 [可并行]
```

---

## 验收标准

- [ ] **自动化验证门禁已通过**：地板规则合规性检查 + 已拆报告存在性检查均无违规
- [ ] 收敛检查清单已完成（依据 convergence_criteria.md）
- [ ] 结论为以下之一：
  - 结论 = "**已收敛**"
  - 生成再入列表，列明具体模块及原因
- [ ] 迭代历史已更新
- [ ] 反懈怠检查：P2 评级已与客观指标交叉验证（结合自动化脚本输出）

---

## 输出

| 产出物 | 说明 |
|---|---|
| 收敛结论 | "已收敛" 或 "未收敛" |
| 再入列表（如适用） | 需进一步分解的模块及原因 |

---

## 跳转

| 结果 | 下一步 |
|---|---|
| 已收敛 | → P4 深度分析 |
| 未收敛 | → P1（携带再入列表，独立模块可并行处理） |
| 已达迭代上限 | → P4（附注"已达迭代上限"） |
