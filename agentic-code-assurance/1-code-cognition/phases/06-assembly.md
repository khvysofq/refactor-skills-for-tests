## 验收标准

### 必须满足

- [ ] **总体报告完整存在**

  ```bash
  [ -f docs/codearch/overall_report.md ] && echo "PASS" || echo "FAIL"
  ```

- [ ] **总体报告包含**：
  - [ ] 工程概览（目标、输入、输出、主流程）
  - [ ] 模块索引：模块依赖图（Mermaid）、模块列表（含状态列）、跨模块关系摘要
  - [ ] 构建与测试摘要（或到 `build_and_tests.md` 的链接）

- [ ] **引用校验**：总体报告中所有 `modules/*.md` 链接均指向实际存在的文件

  ```bash
  grep -oE 'modules/[^)]+\.md' docs/codearch/overall_report.md | sort -u | while read p; do
    [ -f "docs/codearch/$p" ] && echo "OK $p" || echo "BROKEN $p"
  done
  ```

- [ ] **L2 报告抽查**：抽查 2–5 份模块报告，确认包含全部必填章节（职责、边界、依赖、复杂度评级、关键设计要点、代码特征、关键代码位置索引、关键数据流路径、与其它模块的关系、使用示例）

- [ ] **模块索引一致性**：总体报告中的模块索引与 `docs/codearch/modules/` 目录下的实际文件一致

- [ ] **技术特征统计**：总体报告中的技术特征统计为最新状态
