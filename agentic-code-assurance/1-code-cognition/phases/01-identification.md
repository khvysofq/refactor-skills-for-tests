# Phase 01: 模块识别与初步划分

> **阶段代号**: P1  
> **入口条件**: P0 已完成，或从 P3 收敛评估触发重新进入  
> **预计耗时**: 小型项目 <30 分钟 · 中型 30–60 分钟 · 大型 1–2 小时

---

## 目标

识别模块边界，为每个模块创建 L1 报告。

---

## 进入模式

| 模式 | 输入 |
|---|---|
| **首次执行** | engineering_metadata + 整个项目范围 |
| **迭代重入** | engineering_metadata + P3 再入列表中指定的高复杂度模块 |

---

## 执行

加载并执行 [Skill 01 — 模块识别](../skills/skill-01-identify.md)。

---

## 验收标准

- [ ] 每个模块的 L1 报告已存放于 `docs/codearch/modules/`
- [ ] 总体报告的模块索引已更新，包含新识别的模块
- [ ] 每份 L1 报告包含以下内容：
  - [ ] 模块名称
  - [ ] 路径
  - [ ] 代码指标（code metrics）
  - [ ] 职责描述（responsibility）
  - [ ] 依赖概览（dependency overview）

---

## 输出

| 产出物 | 路径 |
|---|---|
| L1 模块报告（每模块一份） | `docs/codearch/modules/<module>.md` |
| 更新后的模块索引 | 总体报告中的模块索引部分 |

---

## 跳转

| 结果 | 下一步 |
|---|---|
| 验收通过 | → P2 分析与复杂度评级 |
| 验收不通过 | → 修复后重新验收 |

---

## 备注

> [可并行] 多个子模块的扫描可并行执行（特别是迭代重入时，P3 识别出的多个高复杂度模块可并行进入 P1）。
