# BUG 确认与修复工作流 - Agent 入口

> **读取指令**：本阶段为「BUG 确认与修复」，根据任务列表编写/运行测试、确认 BUG 存在性、实施修复并进行回归验证。执行前请先阅读根目录 [README.md](../README.md)，确认阶段二已产出任务列表。

---

## 输入

| 输入 | 路径/说明 |
|------|------------|
| 疑似 BUG 任务列表 | `docs/risk_tasks/` 下的任务列表文件 |
| 源代码 | 仓库根目录下的 C/C++ 源码 |
| 构建与测试说明 | `docs/codearch/build_and_tests.md`；按需可参考 `overall_report.md` 与 `modules/<module_name>.md` |

---

## 输出

| 产出 | 说明 |
|------|------|
| 测试用例 | 落入工程现有测试目录，或按本工作流后续约定的位置；用于验证 BUG 存在性及回归。 |
| 修复补丁或变更说明 | 对源码的修复；形式（patch 文件、分支、PR 描述等）由后续工作流约定。 |
| 修复摘要（可选） | `docs/remediation/` 下记录修复摘要（如 `remediation_log.md`），具体格式后续定义。 |

---

## 建议执行顺序

1. 阅读根目录 [README.md](../README.md) 与 `docs/risk_tasks/` 中的任务列表。
2. 对每项任务：结合 `docs/codearch/build_and_tests.md` 构建测试场景（编写或运行测试）。
3. 运行测试，验证 BUG 是否可复现（确认存在性）。
4. 实施修复。
5. 执行回归验证（如运行现有测试套件）。
6. 若在过程中发现工程理解文档与代码不符，按根 README 中的「反馈机制」更新 `docs/codearch/` 下相应报告或模块文档。
